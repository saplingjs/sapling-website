A minimalist framework for faster-than-light development.

Sapling is a Node.js framework for building websites, web apps and APIs as fast as you can imagine them. With unrivalled speed of development, you can prototype quickly, iterate with ease, and deploy the same day. Zero code, zero config – unless you want to. And what's more, it comes with all the SaaS features you never want to write, all built-in.

Your grave concerns.

Rightly you might feel skeptical. Here are the common "but"s and our take on them.


How can all the forms work with no backend code?

Sapling's dynamic data backend figures out what to do based on the type of request it receives to the built-in /data route. It acts as a very light wrapper for abstract CRUD queries;

  • GET /data/posts returns all records in the collection posts
  • GET /data/posts/type/text returns records in the collection posts, where the value of type is text
  • POST /data/posts creates a new record in the collection posts
  • POST /data/posts/_id/1 modifies records in the collection posts, where the value of _id is 1
  • DELETE /data/posts deletes all records in the collection posts
  • DELETE /data/posts/_id/1 deletes records in the collection posts, where the value of _id is 1

As such, the second parameter is the collection, and optionally a third and fourth parameter may be provided, which will act as key and value for a where clause, respectively. The type of HTTP request received determines what is done with the returned records. In cases of POST requests, any and all POST values sent are saved into a new record or any matching records.

Learn more about the data API

Oh wow. Putting this on production would be insanity!

Absolutely, and the default configuration of Sapling is indeed not meant for production use, but rather, optimised for rapid prototyping.

Sapling has robust support for defining data models (aka database schemata) with built-in data validation, and access control for each method-route combination. In the default configuration, these are optional for the developer to define. However, in Sapling's production mode, both of these must be defined, and nothing is ever implicitly saved into the database.

Learn more about production mode

This must be basically a drag'n'drop framework for beginners, with no option for real development.

Just because you don't need to write code to achieve basic CRUD functionality with Sapling, doesn't mean you can't write code when you actually need to.

Sapling has built-in support for custom hooks for any route, including the /data backend. These hooks are just regular Node.js files, where you can code and import whatever you want just like normal, and organise these hooks in any directory structure that makes sense to you. Additionally (and completely optionally), all of Sapling's built-in functions are available to you there.

Learn more about hooks

I don't like MongoDB or Nunjucks or Vue.

You don't have to! By default, Sapling uses these three, but that's all they are – defaults. Each functionality (database, templating, frontend framework) is its own separate package, which you can replace or uninstall at any time.

Database and templating engines connect to Sapling via a generalised abstraction layer called a driver. You can easily write your own driver for your favourite database or templating engine.

Learn more about drivers

What about performance?

Sapling features a production mode that turns off the slowest development aids to bring the end-user performance to the same level as other Node.js based frameworks.

Quick start.

Install the CLI

Run npm install -g @sapling/cli to install the command line tool globally. The command line tool helps you create and manage Sapling projects.

Create your first project

Run sapling create to run a wizard for creating a new blank project. It will automatically create the files and folders needed, and install any dependencies.

Start developing

Navigate to the folder and run sapling run to turn on the server. Read the documentation to learn the basics of Sapling.
